🔍 Dynamic Anomaly Scoring Logic

Traditional systems fail to catch modern UPI frauds like mule account layering and phishing sweeps because they only monitor aggregate volumes. UPI Shield conducts deep-level inspection:

  • Balance Consistency: Discrepancies between initial account balances, final values, and declared transaction amounts are heavily penalized.
  • Temporal Constraints: High frequency transactions occurring during night hours (22:00 to 05:00) increase probability index scales.
  • Explainable Predictions: View precise, detailed feature contribution breakdowns for every verdict rendered.
Detection Algorithm V2 
def evaluate_upi_risk(tx):
    score = 0.0
    # Check Math discrepancy
    diff = tx.bal_before - tx.bal_after
    if abs(diff - tx.amount) > 5.0:
        score += 0.40  # Anomaly
    return score
